BUG 26: Trial Routes Missing Auth Middleware
─────────────────────────────────────────────────────────────────────────────
Status: ⚠️ ISSUE FOUND (Logic Bug, Not Critical)

Evidence:
  File: server/app.js (lines 129-137)
  
  Code:
    const { auth: trialAuth } = require('./middleware/auth');
    app.use('/api/trial', (req, res, next) => {
      // Apply auth middleware only to POST routes (upgrade needs auth, status is public)
      if (req.method === 'POST') {
        return trialAuth(req, res, next);
      }
      // For GET, try to extract user but don't block if no token
      trialAuth(req, res, () => next());
    }, require('./routes/trial').router);

Problem Analysis:
  The code on line 136 attempts to call:
    trialAuth(req, res, () => next())
  
  But the auth middleware in server/middleware/auth.js (line 5-10):
    function auth(req, res, next) {
      const token = ...
      if (!token) return res.status(401).json({ error: 'No token' }); // <-- ALWAYS blocks
      try { req.user = jwt.verify(token, JWT_SECRET); next(); }
      catch(e) { res.status(401).json({ error: 'Invalid token' }); }
    }
  
  The auth function returns early on line 7 WITHOUT calling the fallback 
  callback. The fallback () => next() is NEVER invoked.

Live Test Result:
  GET /api/trial/status (no auth token)
  Expected: 200 (public endpoint per code comment)
  Actual: 401
  Expected: 401
  
  FAIL: The comment says status is "public" but it returns 401

Backend route in server/routes/trial.js (line 22):
  router.get('/status', async (req, res) => {
    // If no token, return anonymous (no trial)
    if (!req.user) {
      return res.json({
        trial_start_date: null,
        ...
        authenticated: false
      });
    }
    ...
  })
  
  The route EXPECTS !req.user to be possible (anonymous access)
  But the middleware prevents req.user from being optional

Analysis:
  ✅ POST /api/trial/upgrade correctly requires auth (works)
  ✗ GET /api/trial/status should allow public access (broken)
  
  The fix would be to either:
  1. Remove the middleware from GET requests entirely
  2. Create a custom "optional auth" middleware
  3. Handle auth inside the route handler instead of at mount point

       try { req.user = jwt.verify(token, JWT_SECRET); }

     }
     next();
